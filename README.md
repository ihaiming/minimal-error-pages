# 简约白色 HTTP 错误页面

谁都不想在浏览网页时碰到冷冰冰的错误页面。这套极简白色风格的错误页面，就是希望当意外发生时，访客看到的不是刺眼的红色警告，而是一段安静的留白，情绪上能稍微缓一缓，然后顺手回到主页继续逛逛

每一个页面都刻意做得很轻：纤细的数字、浅浅的灰色文字、只有悬停时才会微微变化的按钮。没放任何多余的东西，就是为了让“出错”这件事显得不那么沉重

## ✨ 特点

- **极简白色风格** — 纯白背景，深灰文字，干净克制，适合各类网站
- **全设备适配** — 使用 `clamp()` 弹性字号，从小屏手机到宽屏显示器都能完整显示
- **细腻交互** — 按钮悬停有平滑过渡，分割线随悬停延展，触屏设备无悬停粘连
- **模块统一** — 所有页面共享相同的 HTML 结构和 CSS 设计，方便维护和扩展
- **暗色模式友好** — 在暗色模式下仍然保持柔和的白色背景，不会刺眼

## 📂 文件结构

```

├── 404.html          # 页面未找到
├── 403.html          # 禁止访问
├── 500.html          # 服务器内部错误
├── 401.html          # 未授权
└── README.md

```

## **使用方法**

1. 下载对应的 HTML 文件
2. 将文件放置到你的网站服务器上（例如 Nginx 或 Apache 的错误页面目录）
3. 修改按钮链接 `href="/"` 为你网站的真实首页地址
4. 根据需要自定义页面标题、描述文本或颜色变量
5. 配置 Web 服务器使用这些文件作为自定义错误页面

**Nginx 示例：**
```nginx
error_page 404 /404.html;
error_page 403 /403.html;
error_page 500  /500.html;
error_page 401  /401.html;
location = /404.html { internal; }
location = /403.html { internal; }
location = /500.html { internal; }
location = /401.html { internal; }
```

## 自定义

所有页面的核心样式变量定义在 :root 中，你可以轻松修改：

```css
:root {
    --bg: #ffffff;           /* 背景色 */
    --text: #1a1a1a;         /* 主文字色 */
    --button-hover: #1a1a1a; /* 按钮悬停背景 */
}
```

错误码、标题和描述文本直接修改 HTML 中的对应元素即可
